Login con Supabase Auth

El usuario introduce email y contraseña. La app llama a la API de Supabase Auth que devuelve un JWT firmado con ES256.

  • Llama a supabase.auth.signInWithPassword() → API externa Supabase (Criterio 4)
  • Supabase devuelve JWT firmado con clave privada EC P-256
  • JWT se guarda en AsyncStorage automáticamente mediante persistSession: true
  • Después del login → POST /api/auth/sync al backend para sincronizar el usuario en la BD Java
  • Validación con react-hook-form: email válido, contraseña mínimo 6 caracteres

Registro de nuevo usuario

El registro crea una cuenta en Supabase Auth y sincroniza los datos en la tabla users de PostgreSQL.

  • La contraseña nunca llega al backend Java — la gestiona Supabase directamente
  • Validación doble: frontend (react-hook-form) + BD (CHECK constraints + UNIQUE email)

C1
Criterio 1 — Tipos de datos avanzados de PostgreSQL
Uso de tipos nativos más allá de VARCHAR e INTEGER
Fragmento SQL — supabase-schema.sql
CREATE TABLE users (
  id        UUID PRIMARY KEY
            DEFAULT gen_random_uuid(),
  email     TEXT UNIQUE NOT NULL,
  role      user_role   -- ENUM personalizado
            DEFAULT 'user',
  goals     TEXT[],    -- array nativo
  metadata  JSONB,     -- JSON binario
  created_at TIMESTAMPTZ
             DEFAULT NOW()
);
🔑 UUID como clave primaria
Se usa gen_random_uuid() en lugar de SERIAL. Ventaja: el ID es único globalmente — permite federar datos entre Supabase Auth y la base de datos propia sin colisiones.
📋 TEXT[] — Array nativo
La columna goals TEXT[] almacena múltiples objetivos fitness sin tabla auxiliar. Consultable con operadores de array: WHERE 'Perder peso' = ANY(goals).
🗃️ JSONB — Datos semiestructurados
metadata JSONB y exercises JSONB en la tabla workout_sessions guardan el detalle de cada serie (peso/reps) sin necesidad de tabla adicional. Con índice GIN para búsquedas eficientes.

Criterio 5 — Seguridad
JWT ES256 · Spring Security · Row Level Security · Variables de entorno
JwtAuthFilter.java — validación ES256
@Component
public class JwtAuthFilter
    extends OncePerRequestFilter {

  protected void doFilterInternal(...) {
    String token = extractBearer(request);
    // Verifica con clave pública EC P-256
    // de Supabase (no necesita llamada externa)
    Jws<Claims> claims =
      Jwts.parserBuilder()
        .setSigningKey(supabasePublicKey)
        .build()
        .parseClaimsJws(token);
    // Extrae user_id y role del payload
    String userId = claims.getBody()
                          .getSubject();
  }
}
Row Level Security en PostgreSQL
-- Activar RLS en la tabla
ALTER TABLE workout_sessions
  ENABLE ROW LEVEL SECURITY;

-- Política: solo ves tus sesiones
CREATE POLICY sessions_isolation
  ON workout_sessions
  USING (user_id = auth.uid());

-- Entrenadores ven las de sus clientes
CREATE POLICY trainer_view
  ON workout_sessions
  USING (user_id IN (
    SELECT id FROM users
    WHERE trainer_id = auth.uid()
  ));
